class MLP(nn.Module):
    def __init__(self,input_size,hidden_size, output_size):
        super(MLP,self).__init__()
        self.fc1 = nn.Linear(input_size,hidden_size)
        self.tanh1 = nn.Tanh()
        self.fc2 = nn.Linear(hidden_size,hidden_size)
        self.tanh2 = nn.Tanh()
        self.fc3 = nn.Linear(hidden_size,output_size)
    
    def forward(self, x):
       
        x = x.view(-1, 28*28)
        out = self.fc1(x)
        out = self.tanh1(out)
        out = self.fc2(out)
        out = self.tanh2(out)
        out = self.fc3(out)
        return out

history_norm = {'loss': [], 'accuracy': []}
total_step = len(train_loader)

print("\n--- INICIANDO TREINAMENTO COM DADOS NORMALIZADOS ---")
mod.to(device)
mod.train()
for epoch in range(EPOCHS):
    total_loss = 0
    correct = 0
    total = 0
    for i, (images, labels) in enumerate(train_loader):
        # Forward pass
        images = images.to(device)
        labels = labels.to(device)
        outputs = mod(images)
        loss = loss_func(outputs, labels)

        # Backward and optimize
        opt.zero_grad()
        loss.backward()
        opt.step()

        # Monitoramento
        total_loss += loss.item()
        _, predicted = torch.max(outputs.data, 1)
        total += labels.size(0)
        correct += (predicted == labels).sum().item()

    epoch_loss = total_loss / total_step
    epoch_acc = 100 * correct / total
    history_norm['loss'].append(epoch_loss)
    history_norm['accuracy'].append(epoch_acc)

    print(f'poca [{epoch+1}/{EPOCHS}], Perda: {epoch_loss:.4f}, Acurcia: {epoch_acc:.2f}%')

print("--- TREINAMENTO CONCLUDO ---")

Relatrio de Anlise e Comparao de Modelos MLP

1. Introduo

Este relatrio apresenta uma anlise comparativa do desempenho de dois modelos de Perceptron de Mltiplas Camadas (MLP) para uma tarefa de classificao. O objetivo principal avaliar o impacto da normalizao dos dados de entrada no processo de treinamento e na performance final do modelo. Ambos os modelos foram treinados com a mesma arquitetura e os mesmos hiperparmetros, variando apenas o pr-processamento dos dados.

2. Arquitetura do Modelo

A mesma arquitetura de rede neural foi utilizada para ambos os experimentos. A estrutura do MLP composta por:

  • Camada de Entrada: Uma camada linear que transforma o vetor de entrada em um espao de features de dimenso intermediria.
  • Primeira Camada Oculta: Uma camada com neurnios e funo de ativao Tangente Hiperblica (Tanh).
  • Segunda Camada Oculta: Uma segunda camada com a mesma estrutura da anterior.
  • Camada de Sada: Uma camada linear final que mapeia o espao de features para as classes de sada.

Resumo da Arquitetura: Input -> Linear -> Tanh -> Linear -> Tanh -> Linear -> Output

3. Hiperparmetros de Treinamento

Os seguintes hiperparmetros foram mantidos constantes durante o treinamento de ambos os modelos para garantir uma comparao justa:

  • Otimizador: Adam
  • Taxa de Aprendizado (Learning Rate): 0.001
  • Tamanho do Lote (Batch Size): 128
  • Nmero de pocas (Epochs): 5

4. Anlise dos Resultados

4.1. Modelo 1: Treinado com Dados Normalizados

Neste cenrio, os dados de entrada foram pr-processados e escalonados para um intervalo menor.

  • Desempenho no Treinamento: O modelo demonstrou um aprendizado eficiente, atingindo um alto desempenho ao final das pocas de treinamento.
  • Acurcia Final no Conjunto de Teste: Alta

5. Comparao e a Importncia da Normalizao

A comparao direta entre os dois modelos revela a superioridade inequvoca da abordagem com dados normalizados. O primeiro modelo demonstrou ter maior capacidade de aprendizado e, mais importante, generalizou de forma muito mais eficaz para os dados de teste. O segundo modelo, por sua vez, mostrou-se menos capaz de aprender e generalizar, evidenciando a importncia do pr-processamento.

Por que a normalizao to importante?

  • Convergncia Mais Rpida: Otimizadores funcionam melhor quando os dados de entrada esto em uma escala semelhante, permitindo que o modelo aprenda de forma mais rpida e estvel.
  • Estabilidade Numrica: Manter os valores de entrada em um intervalo pequeno ajuda a manter as ativaes e gradientes sob controle, garantindo um processo de treinamento mais estvel e evitando problemas numricos.
  • Eficincia das Funes de Ativao: A normalizao ajuda a manter as entradas em uma regio onde as funes de ativao so mais sensveis, evitando a saturao dos neurnios e permitindo que o aprendizado flua adequadamente por toda a rede.

6. Concluso

Os resultados demonstram que a normalizao dos dados um passo de pr-processamento fundamental. O modelo treinado com dados normalizados apresentou uma capacidade de generalizao significativamente superior, resultando em um modelo final mais robusto e confivel.
