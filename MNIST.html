class MLP(nn.Module):
    def __init__(self,input_size,hidden_size, output_size):
        super(MLP,self).__init__()
        self.fc1 = nn.Linear(input_size,hidden_size)
        self.tanh1 = nn.Tanh()
        self.fc2 = nn.Linear(hidden_size,hidden_size)
        self.tanh2 = nn.Tanh()
        self.fc3 = nn.Linear(hidden_size,output_size)
    
    def forward(self, x):
       
        x = x.view(-1, 28*28)
        out = self.fc1(x)
        out = self.tanh1(out)
        out = self.fc2(out)
        out = self.tanh2(out)
        out = self.fc3(out)
        return out

history_norm = {'loss': [], 'accuracy': []}
total_step = len(train_loader)

print("\n--- INICIANDO TREINAMENTO COM DADOS NORMALIZADOS ---")
mod.to(device)
mod.train()
for epoch in range(EPOCHS):
    total_loss = 0
    correct = 0
    total = 0
    for i, (images, labels) in enumerate(train_loader):
        # Forward pass
        images = images.to(device)
        labels = labels.to(device)
        outputs = mod(images)
        loss = loss_func(outputs, labels)

        # Backward and optimize
        opt.zero_grad()
        loss.backward()
        opt.step()

        # Monitoramento
        total_loss += loss.item()
        _, predicted = torch.max(outputs.data, 1)
        total += labels.size(0)
        correct += (predicted == labels).sum().item()

    epoch_loss = total_loss / total_step
    epoch_acc = 100 * correct / total
    history_norm['loss'].append(epoch_loss)
    history_norm['accuracy'].append(epoch_acc)

    print(f'Época [{epoch+1}/{EPOCHS}], Perda: {epoch_loss:.4f}, Acurácia: {epoch_acc:.2f}%')

print("--- TREINAMENTO CONCLUÍDO ---")

Relatório de Análise e Comparação de Modelos MLP¶

1. Introdução¶

Este relatório apresenta uma análise comparativa do desempenho de dois modelos de Perceptron de Múltiplas Camadas (MLP) para uma tarefa de classificação. O objetivo principal é avaliar o impacto da normalização dos dados de entrada no processo de treinamento e na performance final do modelo. Ambos os modelos foram treinados com a mesma arquitetura e os mesmos hiperparâmetros, variando apenas o pré-processamento dos dados.

2. Arquitetura do Modelo¶

A mesma arquitetura de rede neural foi utilizada para ambos os experimentos. A estrutura do MLP é composta por:

  • Camada de Entrada: Uma camada linear que transforma o vetor de entrada em um espaço de features de dimensão intermediária.
  • Primeira Camada Oculta: Uma camada com neurônios e função de ativação Tangente Hiperbólica (Tanh).
  • Segunda Camada Oculta: Uma segunda camada com a mesma estrutura da anterior.
  • Camada de Saída: Uma camada linear final que mapeia o espaço de features para as classes de saída.

Resumo da Arquitetura: Input -> Linear -> Tanh -> Linear -> Tanh -> Linear -> Output

3. Hiperparâmetros de Treinamento¶

Os seguintes hiperparâmetros foram mantidos constantes durante o treinamento de ambos os modelos para garantir uma comparação justa:

  • Otimizador: Adam
  • Taxa de Aprendizado (Learning Rate): 0.001
  • Tamanho do Lote (Batch Size): 128
  • Número de Épocas (Epochs): 5

4. Análise dos Resultados¶

4.1. Modelo 1: Treinado com Dados Normalizados¶

Neste cenário, os dados de entrada foram pré-processados e escalonados para um intervalo menor.

  • Desempenho no Treinamento: O modelo demonstrou um aprendizado eficiente, atingindo um alto desempenho ao final das épocas de treinamento.
  • Acurácia Final no Conjunto de Teste: Alta

5. Comparação e a Importância da Normalização¶

A comparação direta entre os dois modelos revela a superioridade inequívoca da abordagem com dados normalizados. O primeiro modelo demonstrou ter maior capacidade de aprendizado e, mais importante, generalizou de forma muito mais eficaz para os dados de teste. O segundo modelo, por sua vez, mostrou-se menos capaz de aprender e generalizar, evidenciando a importância do pré-processamento.

Por que a normalização é tão importante?

  • Convergência Mais Rápida: Otimizadores funcionam melhor quando os dados de entrada estão em uma escala semelhante, permitindo que o modelo aprenda de forma mais rápida e estável.
  • Estabilidade Numérica: Manter os valores de entrada em um intervalo pequeno ajuda a manter as ativações e gradientes sob controle, garantindo um processo de treinamento mais estável e evitando problemas numéricos.
  • Eficiência das Funções de Ativação: A normalização ajuda a manter as entradas em uma região onde as funções de ativação são mais sensíveis, evitando a saturação dos neurônios e permitindo que o aprendizado flua adequadamente por toda a rede.

6. Conclusão¶

Os resultados demonstram que a normalização dos dados é um passo de pré-processamento fundamental. O modelo treinado com dados normalizados apresentou uma capacidade de generalização significativamente superior, resultando em um modelo final mais robusto e confiável.
